Why we're here

The Quoting Problem

Without a shared framework, every quote is a custom negotiation — creating inconsistency for sales and scope creep for dev.

⏱️
Dev Time is Unpredictable
A 3-node webhook and a 20-node AI loop both get called "an automation." Without a tier system there's no anchor for dev hour estimates before quoting starts.
📊
Execution Count is Invisible
Scheduled workflows fire 1,440× a month even when there's nothing to process. Event-based workflows only run when triggered. These are totally different cost profiles that look the same on paper.
💸
Tooling Costs Are Consumption-Based
OpenAI, DocuSign, SerpAPI — all billed per use. The price depends on volume, model choice, and token length. None of that is knowable until the workflow is built and running.
Underquoting on complex projects
Overquoting on simple ones (losing deals)
No clear client expectation on ongoing costs

Model Selection — Why Each Was Chosen
Model selection is a deliberate cost-optimization layer. Not every task needs the flagship model.
ModelUsed InReason for Choice
GPT-5.4 (flagship) Tornatech: Whitepaper, Case Study, Social Post Client-facing NFPA 20 technical content — requires accuracy, structure, and consistent tone. Quality over cost.
GPT-5.4-mini CMCA: P0 Sophia, P3+P4+P5 article drafts + refinement; Tornatech: FR + ES translation High-volume internal processing where speed and cost matter more than absolute quality. Still produces excellent output for SEO content and translation.
GPT-5.4-nano Tornatech: Yoast SEO field translation Translating title + meta description only (~200 tokens). Nano is sufficient; using mini here would be wasted spend.
Gemini Flash Lite CPA: Exam document analysis Multimodal PDF/image analysis. Google Gemini is better suited for structured document extraction. Currently in preview (near-zero billing).
Takeaway: At 5× current client volume, total LLM costs would be ~$40–100/month. The dominant ongoing cost is subscriptions (n8n Cloud Pro ~$60, OpenAI API minimum), not per-token usage. LLM cost is predictable and manageable.
